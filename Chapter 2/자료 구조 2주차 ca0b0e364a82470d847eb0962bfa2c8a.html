알고리즘 분석

* 프로파일링

  1. 정의
  • 어떤 응용프로그램에 어느 클래스가 더 좋을지 결정 할 때, 여러가지를 다 시도해보고 각각 얼마나 걸리는지 알아보는 것

2. 특징 :

  • 사전에 모두 구현해봐야 함
  • 컴퓨터의 성능에 따라 결과가 달라질 수 있음
  • 입력으로 사용하는 데이터에 의존하기도 함

* 알고리즘 분석

  • 프로파일링의 문제를 해결할 수 있음
  • 컴퓨터의 성능에 구애받지 않기 위해 각 알고리즘에 필요한 연산 수를 셈
  • 입력 데이터에 의존하지 않으려면 입력 데이터에 대한 평균 성능을 분석할 수 있음
  • 한 알고리즘이 작은 문제에서는 최상의 성능을 보이지만 큰 문제에서는 다른 알고리즘이 좋을 수 있음. 보통은 큰 문제에 초점을 맞춤
    • Ex) 알고리즘 A의 실행시간 : 입력 n의 크기에 비례, B의 실행시간 : n^2에 비례
    • 적어도 n의 값이 클 때는 A가 B보다 빠르다고 간주
    • 상수 시간 : 입력 크기에 의존 X
    • 선형 : 실행시간이 입력 크기에 비례할 경우
    • 이차 : 실행시간이 n^2에 비례할 경우

* Big O 표기법

  1. 정의 : 알고리즘 상황에서 최악의 상황일 때의 시간 복잡도 표기법
  • O(1) < O(log n) < O(n) < O(nlog n) < O(n^2) < O(2^n)

2. 예

  • O(1) : 스택에서 push, pop
  • O(lognlognlogn﻿) : 이진 트리
  • O(nnn﻿) : for문
  • O(nlognnlognnlogn﻿) : quick sort, merge sort, heap sort
  • O(n2n^2n2﻿) : 이중 for문, 삽입 정렬, 버블 정렬, 선택 정렬
  • O(2n2^n2n﻿) : 피보나치 수열

* 정렬 알고리즘으로 보는 시간 복잡도

  1. 스택의 push or pop
    • LIFO 구조로, 가장 마지막 인덱스에 접근하여 값을 넣거나 빼므로 연산이 필요없음

  1. for문
    • 반복되는 횟수만큼 시간이 늘어나므로 O(n)

  1. 퀵 정렬 (Quick Sort)
    • 분할 정복 알고리즘 (Devide & Conquer)
    • pivot을 기준으로 pivot보다 작은 값은 왼쪽, pivot보다 큰 값은 오른쪽으로 옮기면서 정렬함
    • 최선의 경우 : 항상 절반씩 분해가 되는 경우, O(nlog2nnlog_2nnlog2​n﻿)의 시간 복잡도를 가짐

      데이터를 정렬 시,

      • 1회차에 2개로 나눠짐
      • 2회차에 4개로 나눠짐
      • 3회차에 8개로 나눠짐

      ∴ k=log2nk = log_2nk=log2​n﻿ (k = 둘로 나뉘는 횟수, n = 데이터의 개수)

    • 최악의 경우 : 항상 한쪽은 0개, 다른 한쪽은 n-1개로 분해되는 경우, O(n2n^2n2﻿)의 시간복잡도를 가짐
    • 보통 Big O 표기는 최악의 경우를 기준으로 이야기하지만, 퀵 정렬의 경우는 최선의 경우로 이야기 함
    #include <iostream>

using namespace std;

int data[9] = {5, 1, 3, 7, 9, 2, 4, 6, 8};

void swap(int low, int high) {
    int temp = data[low];
    data[low] = data[high];
    data[high] = temp;
}

int partition(int left, int right) {
    int pivot = data[left];
    int low = left + 1;
    int high = right;

    while (low <= high) {
        
        //pivot보다 큰 값
        while( pivot > data[low]) low++;

        //pivot보다 작은 값
        while (pivot < data[high]) high--;

        if (low <= high) {
            swap(low, high);
        }
    }

    swap(left, high);
    return high;
}

void quickSort(int left, int right) {
    if (left <= right) {
        int pivot = partition(left, right);
        quickSort(left, pivot-1);
        quickSort(pivot+1, right);
    }
}

int main() {

    quickSort(0, sizeof(data) / sizeof(int)-1);

    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        cout << data[i];
    }
    return 0;
}

  1. 합병 정렬 (Merge Sort)
    • 분할 정복 알고리즘 (Device & Conquer)
    • 최소 단위로 데이터를 분리 후, merge하면서 정렬하는 알고리즘
    • O(nlog2nnlog_2nnlog2​n﻿)의 시간복잡도를 가짐
      • merge 과정에서 최대 비교연산 수 : lognlognlogn﻿
    #include <iostream>

using namespace std;

int data[9] = {5, 1, 3, 7, 9, 2, 4, 6, 8};

void merge(int left, int mid, int right) {
    int leftIndex = left;
    int rightIndex = mid+1;

    int *sortData = new int[sizeof(int) * (right + 1)];

    int sortIndex = left;

    while (leftIndex <= mid && rightIndex <= right) {
        if (data[leftIndex] <= data[rightIndex])
            sortData[sortIndex] = data[leftIndex++];
        else
            sortData[sortIndex] = data[rightIndex++];

        sortIndex++;
    }

    if (leftIndex > mid) {
        for (int i = rightIndex; i <= right; i++, sortIndex++) {
            sortData[sortIndex] = data[i];
        }
    } else {
        for (int i = leftIndex; i <= mid; i++, sortIndex++) {
            sortData[sortIndex] = data[i];
        }
    }

    for (int i = left; i <= right; i++)
        data[i] = sortData[i];

    delete[] sortData;

}

void mergeSort(int left, int right) {
    int mid;

    if (left < right) {
        mid = (left + right) / 2;

        mergeSort(left, mid);
        mergeSort(mid+1, right);

        merge(left, mid, right);
    }
}

int main() {

    mergeSort(0, sizeof(data)/sizeof(int)-1);

    for (int i = 0; i < sizeof(data)/sizeof(int); i++) {
        cout << data[i];
    }
    return 0;
}

  1. 삽입 정렬 (Insertion Sort)
    • key 인덱스를 기준으로 key 인덱스보다 작은 데이터들을 비교하여, key 인덱스 값보다 작은 값이 나타나면 그 자리에 key 인덱스 값을 꽂아넣으면서 정렬하는 알고리즘
    • O(n2n^2n2﻿)의 시간복잡도를 가짐
    #include <iostream>

using namespace std;

int data[9] = {5, 1, 3, 7, 9, 2, 4, 6, 8};

void insertionSort() {
    int key;
    int i, j;
    for (i = 1; i < sizeof(data) / sizeof(int); i++) {
        key = data[i];
        for (j = i-1; j >= 0; j--) {
            if (key < data[j]) 
                data[j+1] = data[j];
            else
                break;
        }
        data[j+1] = key;
    }
}

int main() {
    insertionSort();

    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        cout << data[i];
    }
    return 0;
}

  1. 버블 정렬 (Bubble Sort)
    • 인접한 두 원소를 비교하여 정렬하는 알고리즘
    • O(n2n^2n2﻿)의 시간복잡도를 가짐
    #include <iostream>

using namespace std;

int data[9] = {5, 1, 3, 7, 9, 2, 4, 6, 8};

void swap (int idx1, int idx2) {
    int temp = data[idx1];
    data[idx1] = data[idx2];
    data[idx2] = temp;
}

void bubbleSort() {
    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        for (int j = i; j < sizeof(data) / sizeof(int) - 1; j++) {
            if (data[j] > data[j+1]) swap(j, j+1);
        }
    }
}

int main() {
    bubbleSort();

    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        cout << data[i];
    }
    return 0;
}

  1. 선택 정렬 (Selection Sort)
    • 데이터가 n개일 때, 특정 인덱스 k를 기준으로 k+1 ~ n-1 인덱스에서 가장 작은 데이터를 찾은 후, k번째 인덱스의 값과 가장 작은 값을 교환하면서 정렬하는 알고리즘
    • O(n2n^2n2﻿)의 시간복잡도를 가짐
    #include <iostream>

using namespace std;

int data[9] = {5, 1, 3, 7, 9, 2, 4, 6, 8};

void swap (int idx1, int idx2) {
    int temp = data[idx1];
    data[idx1] = data[idx2];
    data[idx2] = temp;
}

void selectionSort() {
    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        int minIndex = i;
        for (int j = i+1; j < sizeof(data) / sizeof(int); j++) {
            if (data[j] < data[minIndex])
                minIndex = j;
        }
        swap(i, minIndex);
    }
}

int main() {

    selectionSort();

    for (int i = 0; i < sizeof(data) / sizeof(int); i++) {
        cout << data[i];
    }
    return 0;
}
